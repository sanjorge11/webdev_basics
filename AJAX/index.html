-- Introduction

Note: This tutorial is best experienced by running it on a local server. For more info on XMLHttpRequest Object Methods, visit the tutorial link.

AJAX allows you to:

  • Read data from a web server after the page has loaded
  • Update a web page without reloading the page
  • Send data to a web server - in the background

How AJAX works:

  1. An event occurs in a web page (the page is loaded, a button is clicked)
  2. An XMLHttpRequest object is created by JavaScript in browser
  3. The XMLHttpRequest object sends a request to a web server
  4. The server processes the request
  5. The server sends a response back to the web page
  6. The response is read by JavaScript
  7. Proper action (like page update) is performed by JavaScript - an update to DOM without reloading whole page

-- The XMLHttpRequest Object

The central part of AJAX is the XMLHttpRequest object, it is what's used for exchanging data with a server behind the scenes and without reloading the page. All modern browsers have a built-in XMLHttpRequest object.

Note: For security reasons, modern browsers do not allow access across domains. This means that both the web page and the XML file it tries to load, must be located on the same server.

-- AJAX Request

Two examples of HTTP request methods are GET and POST.

GET is simpleer and faster than POST

You use POST requests when

  • A cached file is not an option (update a file or database on the server).
  • Sending a large amount of data to the server (POST has no size limitations).
  • Sending user input (which can contain unknown characters), POST is more robust and secure than GET.
